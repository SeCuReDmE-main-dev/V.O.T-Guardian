## Issues discovered and fixes

- Multi‑stage Dockerfiles are not supported by the E2B template builder.
  - Fix: Convert to single‑stage Dockerfiles for all template variants.
- `import mindsdb` fails if only `mindsdb_sdk` is installed.
  - Fix: Create a lightweight alias package (`mindsdb/__init__.py`) that re‑exports from `mindsdb_sdk` and exposes `__version__`.
  - Implementation trick: Generate the alias via a base64‑encoded inline Python snippet during the Docker build to avoid quoting/heredoc issues and CRLF pitfalls.
- Validation initially failed for mindsdb/whisper due to pointing at an older template ID and missing API key in the shell environment.
  - Fix: Align `.env` `E2B_TEMPLATE_ID` with the newer template image; ensure `E2B_API_KEY` is exported before running validation.

## Validation work

- Added `developpement/tests/validate_full_template.py` to spawn the sandbox using `E2B_TEMPLATE_ID` and probe libraries: torch, librosa, mindsdb, whisper. Prints a JSON status object with `ok` and `version` for each.
- Outcome (Oct 24, evening): initial runs highlighted missing MindsDB/Whisper.
- Follow‑up (after midnight, Oct 25): with the mid template ID updated and alias in place, validation returned all OK with versions (see separate Oct 25 worklog for exact JSON).
