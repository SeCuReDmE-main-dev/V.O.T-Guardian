## How to rerun validation (Windows PowerShell)

```powershell
# From repo root
$env:E2B_API_KEY = (
  (Get-Content -Raw 'developpement\/.env') -split "`n"
  | Where-Object { $_ -match '^E2B_API_KEY=(.+)$' }
  | ForEach-Object { ($_.Split('=',2))[1].Trim() }
)

# Optionally set/override the template ID explicitly
$env:E2B_TEMPLATE_ID = 'qdxbvb9ya4rw7bcupnok'

# Run validation
python developpement\tests\validate_full_template.py
```

## Notes

- The base64 inline Python approach avoids heredoc and quoting pitfalls and is resilient to CRLF line endings in Dockerfiles.
- Keeping `.env` in sync with the intended template ID is critical for accurate validation.
